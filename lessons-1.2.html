УРОК 2 - Настройка управления

Для начало нам нужно установить несколько переменных движение.

Значение оси и масштаб ввода


Перед заданием полей Scale нам нужно больше узнать о том, как работать со значениями осей.

Значение оси — это численное значение, определяемое типом ввода и способом его использования. При нажатии на кнопки и клавиши на выход подаётся 1. Стики имеют выходные значения от -1 до 1, зависящие от направления и наклона стика.


Рисунок 1 - Ось Y



Мы можем использовать значение оси для управления скоростью. Например, если мы нажмём стик до упора, то значение оси будет 1. Если нажать наполовину, то значение будет 0.5.

Умножая значение оси на переменную скорости, мы можем регулировать с помощью стика скорость движения.


Рисунок 2 - Ось X


Также значение оси можно использовать для задания направления вдоль оси. Если умножить скорость на положительное значение оси, то мы получим положительное смещение. При использовании отрицательного значения оси получим отрицательное смещение. Прибавляя это смещение к местонахождению Character, мы задаём направление его движения.


Рисунок 3 - События Axis


Клавиши клавиатуры могут подавать на выход только значения 1 или 0, то можно использовать scale для преобразования их в отрицательные числа. Это можно сделать, взяв значение оси и умножив его на масштаб.
Умножив положительное (значение оси) на отрицательный (масштаб), мы получим отрицательное значение.

Задайте масштаб клавиш S и A, нажав на поле Scale и введя -1 [1].

Для того что бы начать настраивать оси движения нужно перейни в Project Settings >> Engine вкладка input Здесь в меню Bindings Есть два действия Action Mappings и Axis Mappings Первое отвичает за одиночное нажатия вторая за оси. Нажимаем на вторую вкладку и добавляем две оси с двумя группами. Назовем первую ось ForwardBackward в группе добовляем две клавиши W и S Где в S указываем значение Scale = -1 анологично для второй оси с клавишами A и D где A указываем Scale =-1 и назовем ее RightLift. Созданим еще две оси для оглядования мышью. Назовем их LookUD и LookRL и выстовим значения Mouse X и Mouse Y Также в Action Mappings добавим одну ось и назовем ее Jump а клавишу назначим Space Bar


Рисунок 4 - Параметры input


Переходим в класс BP_Player и открываем Event Graph Здесь правой кнопкой мыши выводим события настроенных осей. Для движение персонажей нужно добавить следующие функции Add movement input к этой функции нужно дать вектор Get actor Forward(Right) Vector Значение векторов зависит от того куда смотрит игрок. Функции Add controller Yaw/Pitch input нужны для осмотра мышью. Так же к функциям даем значение векторов Axis Value Анологично подключим функцию Jump
